Curso Git

Comandos Git

  • Lista de Comandos
    1. git init

      2. Iniciar un repositorio en git

    2. git status

      3. Muetsra el estado del repositorio, para saber que archivos están agregados, cuales faltan, los commits, etc

    3. git add

      4. Prepara el archivo con sus cambios para agregarlo al repositorio

    4. git rm --cached nombreArchivo

      5. Deja el archivo como estaba antes del git add, es decir lo quita del espacio en memoria impidiendo que se puedan hacer commits

    5. git commit

      6. Actualiza los cambios del archivo en el repositorio

    6. git config --list

      7. Muestra las donfiguraciones de git, entre ellas el nombre del usuario y el email

    7. git commit -m "descripción"

      8. Realiza los cambios en el repositorio

    8. git log nombreArchivo.ext

      9. Vusualiza los commit que se han realizaado al archivo, es decir el hisotiral del repositorio

    9. git show nombreArchivo.ext

      10. Muestra los cambios realizados en el archivo, linea por linea

    10. git checkout

      11. Permite actualizar un archivo que ya se guardó en un repositorio de trabajo en equipo, ejemlo cuando dos personas trabajan ne mismo repositorio y un solo archivo pero la otra perosna quiere descragr dichos cambios

    11. git reset --hard o --soft

      12. Permite retornar a una versión anterior del repositorio.
      soft: Borra el historial y los registros de Git de commits anteriores, pero guarda los cambios en Staging para aplicar las últimas actualizaciones a un nuevo commit.
      hard: Deshace todo, absolutamente todo. Toda la información de los commits y del área de staging se elimina del historial.
      mixed: Borra todo, exactamente todo. Toda la información de los commits y del área de staging se elimina del historial.
      git reset HEAD: El comando git reset saca archivos del área de staging sin borrarlos ni realizar otras acciones. Esto impide que los últimos cambios en estos archivos se envíen al último commit. Podemos incluirlos de nuevo en staging con git add si cambiamos de opinión.
      git reset HEAD es un comando que te permite revertir los cambios que ya habías preparado para subir, y moverlos de vuelta a tu proyecto. Con este comando puedes cancelar los cambios que ya habías agregado, para que puedas revisarlos, modificarlos o deshacerlos antes de confirmarlos con un commit.

    12. git rm

      13. Es un comando que nos ayuda a eliminar archivos de Git sin eliminar su historial del sistema de versiones. Para recuperar el archivo eliminado, necesitamos retroceder en la historia del proyecto, recuperar el último commit y obtener la última confirmación antes de la eliminación del archivo.

    13. git clone url

      14. lo que hace es crear una copia del master del repositorio online y lo guarda en el repositorio del usuario

    14. git branch

      15. Permite crear una rama nueva a partir del último commit

    15. git checkout nombreRama

      16. Permite cambiarse entre las ramas existentes

    16. Notas proyecto

      17. Este archivo tiene dos ramas: main y cabecera, tembién permite mostrar las ramas que existen y en cual estoy ubicado actualmente

    17. git merge nombreRamaSecundaria

      18. Este comando permite que se funcionen dos ramas, normalmente se hace el merge desde la principal, entonces el comando sería git merge cabecera, es decir que traigo la rama cabecera que es mi rama nombreRamaSecundaria y la fuciono a la principal que en este caso es main, de esta manera se conserva dicha rama principal

    18. git remote -v

      19. Permite ver los repositorios remotos tanto para fetch como para push

    19. git push

      20. Permite ver los repositorios remotos tanto para fetch como para push

    20. Otros comandos git local y remoto
    21. git log --all --graph --decorate --oneline

      22. Permite ver de forma visual (--graph) los cambios en el proyecto, mostrando las ramas y los comits de cada una de las ramas, merge, etc del histórico del proyecto.

    22. Se crea Alias

      23. Se crea Alias arbol=git log --all --graph --decorate --oneline, este alias se puede usar en la carpeta del proyecto

    23. tags

      24. git tag -a v0.1 -m "Haciendo un merge en el proyecto" 633741b, se crea el tags, el nombre es v0.1 comentario en tre comilllas y la refrencia el commit donde se ubica el tags git show-ref --tags, este comando permite mostrar los tags git push origin --tags, se tiene que hacer push para enviarlo al el repositorio de github
